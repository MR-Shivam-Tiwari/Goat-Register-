## 1. Обзор проекта

**Племенной Реестр Коз** — это официальный племенной реестр для Ассоциации Племенных Коз. Веб-приложение на базе Next.js, позволяющее заводчикам и членам ассоциации:

- Просматривать и искать в реестре всех зарегистрированных коз.
- Просматривать профили хозяйств и их поголовье.
- Просматривать каталог пород (порода → пол → отдельные животные).
- Регистрироваться и входить в систему как заводчик/участник.
- Администраторы могут добавлять коз, редактировать хозяйства и управлять пользователями.
- Просматривать детальные профили животных, включая родословную, потомство и данные о продуктивности.

Система была мигрирована с устаревшего PHP/MySQL приложения. Схема базы данных была сохранена без изменений, поэтому некоторые названия столбцов и хэширование паролей могут показаться необычными.

## 2. Стек технологий

| Слой | Технология |
|---|---|
| Фреймворк | Next.js 15+ (App Router) |
| Язык | TypeScript 5 |
| Стилизация | Tailwind CSS v4 |
| База данных | PostgreSQL (через драйвер `pg`) |
| Хостинг (Frontend) | Vercel |
| Хостинг БД | Supabase (PostgreSQL) |
| Шрифты | Google Fonts: Inter, Outfit |
| Иконки | Lucide React |
| Уведомления | react-hot-toast |
| React | v19+ (React Compiler включен) |

> **Почему без ORM?** Схема была перенесена из старой MySQL базы PHP-сайта. Использование чистого SQL с драйвером `pg` позволило избежать сложностей с маппингом ORM на устаревшую схему. Все запросы находятся в файлах страниц или в `lib/db.ts` / `lib/goats-data.ts`.

## 4. Переменные окружения

Файл: `.env.local` (никогда не фиксируйте этот файл в Git)

```
DATABASE_URL=postgresql://user:password@host:port/dbname?pgbouncer=true&connection_limit=1
```

### Заметки по DATABASE_URL

- Приложение подключается к экземпляру **Supabase** PostgreSQL.
- В продакшене (Vercel) используйте URL **Transaction Pooler** от Supabase (порт 6543), а не прямое подключение (Direct Connection), чтобы избежать проблем с IPv6.
- Строка подключения должна включать `?pgbouncer=true&connection_limit=1` при использовании пулера.
- SSL автоматически включается в продакшене (`rejectUnauthorized: false` установлено в `lib/db.ts`).

## 6. Обзор базы данных

Веб-приложение использует базу данных PostgreSQL с унаследованной схемой. Основные таблицы:

### Основные таблицы
- **`animals`**: Основная таблица реестра (id, name, sex, id_farm, id_mother, id_father, status).
- **`goats_data`**: Расширенные данные животных (порода, племенная книга, коды, владелец, дата рождения).
- **`farms`**: Профили хозяйств (название, описание, фотографии).
- **`users`**: Учетные записи пользователей и роли (логин, пароль, email, роль).
- **`breeds`**: Определения пород (название, алиас).

### Вспомогательные таблицы
- **`goats_lact`**: Записи о лактации (удой, жир, белок, дни).
- **`goats_milk`**: Показатели продуктивности (пиковый удой, лактоза, плотность).
- **`goats_test`**: Экспертные оценки и промеры тела.
- **`goats_cert`**: Связи данных для сертификации (родословные для документов).
- **`goats_pic`**: Галерея изображений животных.
- **`stoodbook`**: Классификация племенных книг (Основная, RCB и т.д.).

## 7. Система аутентификации и сессий

- **Вход**: Обрабатывается через Server Action в `lib/actions.ts` (`loginAction`).
- **Хэширование**: Используется MD5 для совместимости с базой PHP-сайта.
- **Сессия**: На базе куки (`uid_token`).
- **Защита**: `lib/access-control.ts` предоставляет функции `getSessionUser()` и `adminOnly()`.

## 8. Интернационализация (i18n)

Приложение полностью локализовано на **Русский (RU)** и **Английский (EN)** языки.

- **Хранение**: `lib/translations.ts` содержит объект `translations`.
- **Сохранение выбора**: В куки `nxt-lang`.
- **Использование**: Каждый серверный компонент считывает куки и вызывает `getTranslation(lang)`.
- **Переключатель**: Компонент `LanguageSwitcher.tsx` меняет локаль.

## 9. Страницы и роуты

### `/goats/[id]` — Профиль козы
Самая сложная страница, включающая:
- **Шапка**: Кличка, порода, пол, код реестра и быстрые ссылки на родителей.
- **Инфо**: Табличный вид официальных данных реестра.
- **Галерея**: Карусель/сетка фото из `goats_pic`.
- **Родословная**: Интерактивное дерево на 4 поколения.
- **Потомство**: Детальная сводка и таблица прямых потомков.
- **Продуктивность**: Данные о лактациях (молоко, жир, белок, лактоза и т.д.).
- **Экспертная оценка**: Результаты официальных тестов и промеры.
- **Данные для сертификата**: Выбор данных лактации для документов.
- **Родословная 3-го поколения**: Сокращенные показатели предков.
- **Движение**: Лог смены владельцев и хозяйств.

### `/farms/[id]` — Профиль хозяйства
Показывает описание фермы и анализ поголовья (активное vs выбывшее).

## 10. Справочник компонентов

- **`Header.tsx`**: Верхняя панель с логотипом и контактами.
- **`Navbar.tsx`**: Навигационные ссылки + Auth-состояние (Вход/Регистрация или Выход/Профиль).
- **`LanguageSwitcher.tsx`**: Выбор языка (RU/EN).
- **`Breadcrumbs.tsx`**: Путь навигации («хлебные крошки»).
- **`GoatFilters.tsx`**: Расширенные фильтры для основного реестра.
- **`GoatTable.tsx`**: Универсальная таблица для списков животных.
- **`PedigreeNode.tsx` / `PedigreeChart`**: Логика отрисовки древа предков.
- **`InviteSection.tsx`**: Инструментарий для генерации гостевых ссылок.
- **`AddGoatForm.tsx` / `FarmEditor.tsx`**: Формы добавления/редактирования записей.

## 11. Библиотечные модули (lib/)

- **`lib/db.ts`**: Пул подключений PostgreSQL (через `pg`).
- **`lib/goats-data.ts`**: Централизованные запросы данных для коз.
- **`lib/actions.ts`**: Server Actions для авторизации и переключения языка.
- **`lib/access-control.ts`**: Валидация сессий и ролевая защита.
- **`lib/translations.ts`**: Все многоязычные строки интерфейса.
- **`lib/auth.ts`**: Справочник устаревшего хэширования паролей (MD5/CRC32).

---

## 12. Ключевые проектные решения

- **Чистый SQL**: Используется для работы с унаследованной схемой без накладных расходов ORM.
- **Force Dynamic**: Страницы с куки принудительно рендерятся на сервере для корректного языка и авторизации.
- **Tailwind v4**: Использует современные CSS-возможности для премиального внешнего вида.

## 13. Общие задачи поддержки

- **Обновление текста**: Редактируйте `lib/translations.ts`.

## 14. Известные ограничения и TODO

- **Запись в БД**: Формы Коз/Ферм требуют реализации Server Actions для сохранения данных.
- **Хранение файлов**: Загрузка фото использует локальные пути; следует перейти на облачное хранилище (например, Supabase Storage).
- **Безопасность**: Перейти с MD5 на bcrypt для соответствия современным стандартам.
- **Пагинация**: Большие списки требуют реализации постраничного вывода (offset или cursor).
- **Rich Text**: `react-quill-new` установлен, но еще не интегрирован в редактор хозяйств.
- **Hardcoded Phone**: Контактный телефон организации жестко прописан в `Header.tsx`.
